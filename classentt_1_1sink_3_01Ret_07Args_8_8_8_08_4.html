Detailed Description

template<typename Ret, typename... Args>
class entt::sink< Ret(Args...)>

Sink implementation.

A sink is an opaque object used to connect listeners to signals.
The function type for a listener is the one of the signal to which it belongs.

The clear separation between a signal and a sink permits to store the former as private data member without exposing the publish functionality to the users of a class.

Template Parameters
RetReturn type of a function type.
ArgsTypes of arguments of a function type.

template<typename Ret , typename... Args>
template<auto Candidate, typename Type >
void entt::sink< Ret(Args...)>::connect ( Type *  value_or_instance)
inline

Connects a member function or a free function with payload to a signal.

The signal isn't responsible for the connected object or the payload. Users must always guarantee that the lifetime of the instance overcomes the one of the delegate. On the other side, the signal handler performs checks to avoid multiple connections for the same function.
When used to connect a free function with payload, its signature must be such that the instance is the first argument before the ones used to define the delegate itself.
