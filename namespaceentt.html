Connects a dependency function to the given sink.

A dependency function is a built-in listener to use to automatically assign components to an entity when a type has a dependency on some other types.

The following adds components AType and AnotherType whenever MyType is assigned to an entity:

entt::DefaultRegistry registry;
entt::dependency<AType, AnotherType>(registry.construction<MyType>());

◆ ident

template<typename... Types>
constexpr auto entt::ident = internal::Identifier<std::decay_t<Types>...>{std::make_index_sequence<sizeof...(Types)>{}}

Types identifers.

Variable template used to generate identifiers at compile-time for the given types. Use the constexpr get member function to know what's the identifier associated to the specific type.

Note
Identifiers are constant expression and can be used in any context where such an expression is required. As an example:
constexpr auto identifiers = entt::ident<AType, AnotherType>;
switch(aTypeIdentifier) {
case identifers.get<AType>():
// ...
break;
case identifers.get<AnotherType>():
// ...
break;
default:
// ...
}
In case of single type list, get isn't a member function template:
func(std::integral_constant<
entt::ident<AType>::identifier_type,
entt::ident<AType>::get()
>{});
Template Parameters
TypesList of types for which to generate identifiers.
