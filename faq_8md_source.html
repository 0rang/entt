28 ## Why is my debug build on Windows so slow?
29 
30 `EnTT` is an experimental project that I also use to keep me up-to-date with the
31 latest revision of the language and the standard library. For this reason, it's
32 likely that some classes you're working with are using standard containers under
33 the hood.<br/>
34 Unfortunately, it's known that the standard containers aren't particularly
35 performing in debugging (the reasons for this go beyond this document) and are
36 even less so on Windows apparently. Fortunately this can also be mitigated a
37 lot, achieving good results in many cases.
38 
39 First of all, there are two things to do in a Windows project:
40 
41 * Disable the [`/JMC`](https://docs.microsoft.com/cpp/build/reference/jmc)
42  option (_Just My Code_ debugging), available starting in Visual Studio 2017
43  version 15.8.
44 
45 * Set the [`_ITERATOR_DEBUG_LEVEL`](https://docs.microsoft.com/cpp/standard-library/iterator-debug-level)
46  macro to 0. This will disable checked iterators and iterator debugging.
47 
48 Moreover, the macro `ENTT_DISABLE_ASSERT` should be defined to disable internal
49 checks made by `EnTT` in debug. These are asserts introduced to help the users,
50 but require to access to the underlying containers and therefore risk ruining
51 the performance in some cases.
52 
53 With these changes, debug performance should increase enough for most cases. If
54 you want something more, you can can also switch to an optimization level `O0`
55 or preferably `O1`.
